部署静态网站

听说vercel现在被墙了，需要绑定自定义域名，也就是前提要需要拥有一个自己的域名。

第三方导入（利用github）：

在vercel页面中新建项目 通过GitHub仓库进行导入。

直接导入（使用命令部署）

  • vercel 可以用 vc 来代替，vc 是 Vercel 的缩写
  • vercel login：登录 Vercel 账号
  • vercel dev：本地开启服务
  • vercel dev --bug：本地开启服务并打印日志
  • vercel：部署本地资源到 Vercel 上
  • vercel --prod：更新本地网页

墙裂推荐这个，没有git那样多的命令输入，像git add\git commit\git push 等一系列命令。才会导入完成。

会出现的问题

vercel 部署后出现：一般都是没有配置主路由造成的

404: NOT_FOUND;Code: NOT_FOUND

如果您在 Vercel 部署后遇到了 404: NOT_FOUND 错误，这通常意味着 Vercel 在尝试找到您的静态资源时未找到相应的文件或路径。以下是一些建议，帮助您解决这个问题：

  • 配置路由：
    如果您的静态网站使用了前端路由，需要在 Vercel 中配置路由以确保正确的页面导航。您可以在项目根目录下创建一个 vercel.json 文件，并添加路由配置，例如：

  • 检查静态资源路径：
    确保您的静态网站文件都位于正确的路径中。Vercel 默认会查找 public、out、dist 或 build 文件夹作为静态资源的存放位置。如果您的静态资源位于其他路径，可以尝试将其移动到这些默认文件夹中。

  • 检查文件名大小写：
    确保文件名的大小写与路径在代码中引用的大小写一致。在某些操作系统中，文件名大小写敏感，可能会导致找不到文件的问题。

  • 重新部署：
    尝试重新部署项目，有时候重新部署可以解决一些部署问题。
