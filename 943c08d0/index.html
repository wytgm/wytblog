ComfyUI系列（二）：节点使用详解手册

发表于2024-04-01更新于2024-04-01
字数总计:5.2k阅读时长:16分钟阅读量: 山东

ComfyUI节点使用手册
ComfyUI官方使用手册【官网直译+关键补充】

简单使用

此时，您应该已经在浏览器标签页中成功运行了ComfyUI。加载的默认工作流是一个了解基础操作的好起点。您可以按住鼠标左键拖动画布，或按住空格键并移动鼠标来移动画布。您可以通过鼠标滚轮来缩放。

在我们运行默认工作流之前，让我们做一个小修改，以预览生成的图像而不立即保存它们：

右键点击Save Image节点，然后选择Remove。
在画布的空白部分双击，输入preview，然后点击PreviewImg选项。
找到VAE Decode节点的IMAGE输出，并将其连接到您刚添加的PreviewImg节点的images输入上。
这个修改可以在不立即将结果保存到磁盘上的情况下预览您的结果。不用担心，如果您真的喜欢某个特定的结果，您仍然可以右键点击图像并选择Save Image来保存图像。

通过在菜单中点击Queue Prompt，或在键盘上按Cmd+Enter或Ctrl+Enter，来生成您的第一张图像，就是这么简单！

为了便于分享，许多Stable Diffusion的接口，包括ComfyUI，都会在生成的PNG内部存储生成流程的详细信息。许多与ComfyUI相关的工作流指南也会包含这些元数据。要加载生成图像的关联流程，只需通过菜单中的Load按钮加载图像，或将其拖放到ComfyUI窗口中。这将自动解析详细信息并加载所有相关节点，包括它们的设置。

采样器Ksampler

K采样器使用提供的模型以及正向和负向条件来生成给定潜在图像的新版本。首先，根据给定的seed和denoise强度对潜在图像进行加噪，部分擦除潜在图像。然后，使用给定的Model和positive、negative条件作为指导，去除这些噪声，"梦想"在图像被噪声擦除的地方生成新的细节。

  • 输入端:

    • Model：用于去噪的模型
    • Positive：正向提示词。
    • Negative：负向提示词。
    • latent_image：将被去噪的潜在图像。
    • seed：种子生成方式，用于创建噪声的随机种子。
    • control_after_generate：在每个提示后更改上述种子号的能力。节点可以randomize（随机）、increment（根据种子数做增量）、decrement（根据种子数做减量）或fixed（固定种子数）。
    • steps：迭代步数，去噪过程中使用的步骤数。采样器允许进行的步骤越多，结果就越准确。有关如何选择适当步骤数量的好指南，请参见samplers页面。
    • cfg：提示词相关性，数值越大越符合，分类器自由引导（cfg）比例决定了采样器在实现提示内容方面的积极性。更高的比例强制图像更好地代表提示，但设置过高的比例会负面影响图像质量。
    • sampler_name：随机性强的名字：最后带a的和带sde的。使用哪个采样器，有关可用采样器的更多详情，请参见关于Stable Diffusion采样器：综合指南（Stable Diffusion Samplers: A Comprehensive Guide）。
    • scheduler：使用哪种计划，有关可用计划的更多详情，请参见s请参见关于Stable Diffusion采样器：综合指南（Stable Diffusion Samplers: A Comprehensive Guide）。
    • denoise：通过噪声擦除多少潜在图像的信息。

  • 输出

    • LATENT：去噪后的潜在图像。

高级采样器Ksampler(advance)

常用于SDXL版本。高级K采样器节点是 KSampler 节点的更高级版本。KSampler 节点总是向潜在空间添加噪声，然后完全去除噪声，而 KSampler 高级节点提供了额外的设置来控制这种行为。KSampler 高级节点可以通过 add_noise 设置来选择不向潜在空间添加噪声。它还可以通过 return_with_leftover_noise 设置返回部分去噪的图像。与 KSampler 节点不同，这个节点没有 denoise 设置，但这个过程是通过 start_at_step 和 end_at_step 设置来控制的。这使得例如可以将部分去噪的潜在空间交给另一个 KSampler 高级节点来完成处理。

加载器（XXX Loader）

加载器名称（中文）加载器名称（英文）简单介绍
GLIGEN 加载器节点GLIGEN LoaderGLIGEN 加载器节点可用于加载特定的 GLIGEN 模型。GLIGEN 模型用于将空间信息关联到文本提示的部分，指导扩散模型生成遵循 GLIGEN 指定的构图的图像。
unCLIP 检查点加载器节点unCLIP Checkpoint LoaderunCLIP 检查点加载器节点可用于加载专为 unCLIP 设计的扩散模型。unCLIP 扩散模型用于去噪不仅基于提供的文本提示，还基于提供的图像的潜在变量。此节点还将提供适当的 VAE 和 CLIP 以及 CLIP 视觉模型。(尽管此节点可用于加载所有扩散模型，但并非所有扩散模型都与 unCLIP 兼容。)
加载 CLIP 视觉模型节点Load CLIP Vision加载 CLIP 视觉模型节点可用于加载特定的 CLIP 视觉模型，类似于 CLIP 模型用于编码文本提示的方式，CLIP 视觉模型用于编码图像。
加载 CLIP 节点Load CLIP加载 CLIP 节点可用于加载特定的 CLIP 模型，CLIP 模型用于编码指导扩散过程的文本提示。
加载 ControlNet 模型节点Loader ControlNet Model加载 ControlNet 模型节点可用于加载 ControlNet 模型。类似于 CLIP 模型提供文本提示以指导扩散模型的方式，ControlNet 模型用于为扩散模型提供视觉提示。这个过程与例如给扩散模型一个部分加噪的图像进行修改不同。相反，ControlNet 模型可以用来告诉扩散模型例如最终图像中的边缘应该在哪里，或者主体应该如何摆放。此节点也可用于加载 T2IAdaptors。
加载 LoRA 节点Load LoRA加载 LoRA 节点可用于加载 LoRA。LoRAs 用于修改扩散和 CLIP 模型，改变潜变量去噪的方式。典型的用例包括增加模型生成特定风格的能力，或更好地生成特定主题或动作。甚至可以将多个 LoRAs 链接在一起，以进一步修改模型。（LoRA 强度值可以设置为负值。有时这会产生有趣的效果。）
加载 VAE 节点Load VAE加载 VAE 节点可用于加载特定的 VAE 模型，VAE 模型用于将图像编码和解码至潜在空间。尽管 加载检查点 节点提供了一个 VAE 模型以及扩散模型，但有时使用特定的 VAE 模型会更有用。
加载放大模型节点Load Upscale Model加载放大模型节点可用于加载特定的放大模型，放大模型用于放大图像的尺寸。
加载检查点Load Checkpoint加载检查点节点可用于加载扩散模型，扩散模型用于去噪潜变量。此节点还将提供适当的VAE和CLIP模型。
加载风格模型节点Load Style Model加载风格模型节点可用于加载一个风格模型。风格模型可以用来为扩散模型提供一个视觉提示，指明去噪后的潜在变量应该处于什么样的风格。(目前仅支持 T2IAdaptor 风格模型)
超网络加载器节点Hypernetwork Loader超网络加载器节点可用于加载一个超网络。类似于LoRAs，它们用于修改扩散模型，改变潜在变量的去噪方式。典型的用例包括增加模型生成特定风格的能力，或更好地生成特定主题或动作。甚至可以将多个超网络串联起来，进一步修改模型。

Image（图像节点）

ComfyUI 提供了多种节点来操作像素图像。这些节点可用于加载图像以进行 img2img 工作流程、保存结果，或例如放大图像以进行提高图像分辨率工作流程。

加载图像节点（Load Image）

加载图像节点可用于加载图像。可以通过启动文件对话框或将图像拖放到节点上来上传图像。一旦图像被上传，它们就可以在节点内部被选择。

默认情况下，图像将被上传到ComfyUI的输入文件夹中

输入端输出端
image：要使用的图像的名称。IMAGE：像素图像。
MASK：图像的Alpha通道。

为外绘制填充图像节点（Pad Image For Outpainting）

名词解释：图像修补即为Inpainting，意为在图像内部重新绘制缺失的部分。而外绘制即为Outpainting，意为在图像外部绘制新的内容。为外绘制填充图像节点可用于为外绘制添加图像填充。然后，可以通过VAE编码进行修复将此图像提供给一个修复扩散模型。

输入端输出端
image：要填充的图像。IMAGE：填充后的像素图像。
left：图像左侧的填充量。MASK：指示采样器在哪里进行外绘制的遮罩。
top：图像顶部的填充量。
right：图像右侧的填充量。
bottom：图像底部的填充量。
feathering：原始图像边缘的柔化的程度。

预览图像节点（Preview Image）

预览图像节点可用于在节点图内预览图像。输入:image:要预览的像素图像。此节点没有输出。

保存图像节点（Save Image）

保存图像节点可用于保存图像。要在节点图内简单预览图像，请使用预览图像节点。跟踪您生成的所有图像可能会很困难。为了帮助组织您的图像，您可以将特殊格式化的字符串传递给带有file_prefix部件的输出节点。有关如何格式化字符串的更多信息，请参见保存文件格式

输入端，无输出端：

  • image:要预览的像素图像。
  • filename_prefix:要放入文件名中的前缀。

图像反转节点（Invert Image）

反转图像节点可用于反转图像的颜色。

输入端输出端
image：要反转的像素图像。IMAGE：反转后的像素图像。

图像放大

使用模型放大图像节点（Upscale Image Using Model）

使用模型放大图像节点可以用来通过加载的模型放大像素图像，该模型通过加载放大模型节点加载。

输入端输出端
upscale_model：用于放大的模型。IMAGE：放大后的图像。
image：要放大的像素图像。

图像放大节点（Upscale Image）

图像放大节点可用于调整像素图像的大小。要使用AI放大图像，请参见上一节点。

输入端输出端
image：要放大的像素图像。IMAGE：调整大小后的图像。
upscale_method：用于调整大小的方法。
Width：目标宽度（像素）。
height：目标高度（像素）。
crop：是否对图像进行中心裁剪以保持原始潜在图像的宽高比。

后处理节点

图像模糊节点（Image Blur）

图像模糊节点可用于对图像应用高斯模糊。

输入端输出端
image：要模糊的像素图像。IMAGE：模糊后的像素图像。
blur_radius：高斯的半径。
sigma：高斯的sigma值，sigma值越小，核心就越集中于中心像素。

图像锐化节点（Image Sharpen）

图像锐化节点可用于对图像应用拉普拉斯锐化滤波器。

输入端输出端
image：要锐化的像素图像。IMAGE：锐化后的像素图像。
sharpen_radius：锐化核的半径。
sigma：高斯的sigma值，sigma值越小，核心就越集中于中心像素。
alpha：锐化核的强度。

图像混合节点（Image Blend）

图像混合节点可用于将两个图像混合在一起。

如果第二个图像的尺寸与第一个不匹配，则会重新调整尺寸并中心裁剪以保持其纵横比

输入端输出端
image1：第一个像素图像。IMAGE：混合后的像素图像。
image2：第二个像素图像。
blend_factor：第二个图像的不透明度。
blend_mode：图像混合的方式。

图像量化节点（Image Quantize）

图像量化节点可用于量化图像，减少图像中的颜色数量。

输入端输出端
image：要量化的像素图像。IMAGE：量化后的像素图像。
colors：量化图像中的颜色数量。
dither：是否使用抖动使量化图像看起来更平滑。
blend_mode：图像混合的方式。

条件假设节点（Conditioning）

在ComfyUI中，条件假设节点被用来指导扩散模型生成特定的输出。所有的条件假设都是从一个由CLIP通过Clip Text Encode节点嵌入的文本提示开始的。这些条件随后可以通过本段中可以找到的其他节点进一步增强或修改。

例如，使用Conditioning (Set Area)、Conditioning (Set Mask)或GLIGEN Textbox Apply节点指导过程朝着特定的构图发展。

或者通过Apply Style Model、Apply ControlNet或unCLIP Conditioning节点等提供额外的视觉提示。相关节点的完整列表可以在侧边栏中找到。

Apply ControlNet（应用ControlNet节点）

应用ControlNet节点可用于为扩散模型提供进一步的视觉指导。与unCLIP嵌入不同，controlnets和T2I适配器适用于任何模型。通过链接多个节点，可以使用多个controlNets或T2I适配器指导扩散模型。例如，这可以用于通过提供包含边缘检测的图像以及针对边缘检测图像训练的controlNet给该节点，来提示扩散模型在最终图像中边缘的位置。

要使用T2IAdaptor风格模型，请改用应用风格模型节点。

输入端输出端
conditioning：一个条件设置。CONDITIONING：包含control_net和视觉指导的条件设置。
control_net：一个经过训练，用于使用特定图像数据指导扩散模型的controlNet或T2I适配器。
image：用作扩散模型视觉指导的图像。

CLIP 文本编码节点 (CLIP Text Encode (Prompt))

CLIP 文本编码节点可以使用 CLIP 模型对文本提示进行编码，生成一个嵌入，该嵌入可用于指导扩散模型生成特定图像。有关 ComfyUI 中所有文本提示相关功能的完整指南，请参见文本提示。

输入端输出端
clip：用于编码文本的CLIP模型。CONDITIONING：包含用于指导扩散模型的嵌入文本的条件设置。
text：要编码的文本。

CLIP 视觉编码节点（CLIP Vision Encode Node）

CLIP 视觉编码节点可以用来使用 CLIP 视觉模型对图像进行编码，生成一个嵌入，该嵌入可用于指导 unCLIP 扩散模型或作为风格模型的输入。

输入端输出端
clip_vision：用于编码图像的 CLIP 视觉模型。CLIP_VISION_OUTPUT：编码后的图像。
image：要编码的图像。

CLIP 设置最后一层节点 (CLIP Set Last Layer Node)

CLIP 设置最后一层节点 (CLIP Set Last Layer Node)可用于设置CLIP输出层，以获取文本嵌入。文本编码成嵌入是通过文本被CLIP模型中的各个层转换实现的。虽然传统上扩散模型是基于CLIP中最后一层的输出进行条件化的，但一些扩散模型已经在较早的层上进行了条件化，并且在使用最后一层的输出时可能不会工作得那么好。

输入端输出端
clip：用于文本编码的CLIP模型。CLIP：设置了新输出层的CLIP模型。

GLIGEN文本框应用节点（GLIGEN Textbox Apply node）

GLIGEN 文本框应用节点可用于为扩散模型提供进一步的空间指导，引导其在图像的特定区域生成提示的指定部分。虽然文本输入将接受任何文本，但如果输入是文本提示的一部分，则GLIGEN的效果最佳。

ComfyUI中坐标系统的原点位于左上角。

输入端输出端
conditioning_to：一个调节。CONDITIONING：包含GLIGEN和空间指导的调节。
clip：一个CLIP模型。
gligen_textbox_model：一个GLIGEN模型。
text：要关联空间信息的文本。
width：区域的宽度。
height：区域的高度。
x：区域的x坐标。
y：区域的y坐标。

unCLIP条件化节点 (unCLIP Conditioning)

unCLIP条件化节点可用于通过CLIP视觉模型编码的图像为unCLIP模型提供额外的视觉指导。此节点可以串联以提供多个图像作为指导。

并非所有扩散模型都兼容unCLIP条件化。这个节点特别需要一个考虑到unCLIP的扩散模型。

输入端输出端
conditioning：条件化。CONDITIONING：包含额外视觉指导的条件化，用于unCLIP模型。
clip_vision_output：由CLIP视觉模型编码的图像。
strength：unCLIP扩散模型应该被图像指导的强度。
noise_augmentation：噪声增强可以用来指导unCLIP扩散模型到原始CLIP视觉嵌入附近的随机位置，为生成的图像提供与编码图像密切相关的额外变化。

平均调节 (Conditioning (Average)) 节点

平均调节节点可用于根据在conditioning_to_strength中设置的强度因子，对两个文本嵌入进行插值。

输入端输出端
conditioning_to：在conditioning_to_strength为1时的文本嵌入条件。CONDITIONING：基于conditioning_to_strength混合文本嵌入的新条件。
conditioning_from：在conditioning_to_strength为0时的文本嵌入条件。
conditioning_to_strength：混合conditioning_to到conditioning_from的因子。

应用样式模型节点（Apply Style Model node）

应用样式模型节点（Apply Style Model node）可用于为扩散模型提供进一步的视觉指导，特别是关于生成图像的风格。该节点采用T2I风格适配器模型和来自CLIP视觉模型的嵌入，以指导扩散模型朝向CLIP视觉嵌入的图像风格。

输入端输出端
conditioning：一个条件。CONDITIONING：包含T2I风格适配器和朝向所需风格的视觉指导的条件化。
style_model：一个T2I风格适配器。
CLIP_vision_output：包含所需风格的图像，由CLIP视觉模型编码。

调节（合并）节点（Conditioning (Combine)）

调节（合并）节点可用于通过平均扩散模型预测的噪声来合并多个调节。请注意，这与调节（平均）节点不同。这里，基于不同调节（即构成调节的所有部分）的扩散模型输出被平均，而调节（平均）节点则插值存储在调节内的文本嵌入。

尽管调节合并没有因子输入来确定如何插值两个结果噪声预测，但可以使用调节（设置区域）节点在合并它们之前对各个调节进行加权。

输入端输出端
conditioning_1：第一个调节。CONDITIONING：包含两个输入的新调节，稍后将由采样器平均。
conditioning_2：第二个调节。

调节（设置区域）节点（Conditioning (Set Area) node）

调节（设置区域）节点可用于将一个调节限制在图像的指定区域内。与调节（合并）节点一起使用，可以更好地控制最终图像的构成。

ComfyUI中坐标系统的原点位于左上角。

在混合来自扩散模型的多个噪声预测之前，strength会被标准化。

输入端输出端
conditioning：将被限制在一个区域内的调节。CONDITIONING：限制在指定区域的新调节。
width：区域的宽度。
height：区域的高度。
x：区域的x坐标。
y：区域的y坐标。
strength：在混合多个重叠调节时使用的区域权重。

调节（设置遮罩）节点（Conditioning (Set Mask)）

调节（设置遮罩）节点可用于将一个调节限制在指定的遮罩内。与调节（合并）节点一起使用，可以更好地控制最终图像的构成。

在混合来自扩散模型的多个噪声预测之前，strength会被标准化。

输入端输出端
conditioning：将被限制在遮罩内的调节。CONDITIONING：限制在指定遮罩的新调节。
mask：用于限制调节的遮罩。
strength：在混合多个重叠调节时使用的遮罩区域权重。
set_cond_area：是去噪整个区域，还是仅限于遮罩的边界框。

潜在模型（Latent）

潜在扩散模型，如 Stable Diffusion，并不在像素空间中操作，而是在潜在空间中去噪。这些节点提供了使用编码器和解码器在像素空间和潜在空间之间切换的方法，并提供了多种操纵潜在图像的方式。
