部署到GitHub上

前提：要注册一个github账号。

GitHub新建仓库

先在github中新建一个仓库，点击头像，进入仓库，点击新建仓库，进入新建仓库页面。

对新建的仓库进行一些配置：仓库名、简介描述、私有库or公有库。

使用git连接github账号

打开git命令(前提：安装git-bash)，准备连接github账号。git for Windows

访问网址

在github的仓库里，点击setting，左侧的pages就会显示网址。

使用 SSH 连接将 Hexo 部署到 GitHub 后，您可以通过以下网址来访问您的博客页面：https://username.github.io/repository/

其中：

  • username 是您的 GitHub 用户名。
  • repository 是您在 GitHub 上存储 Hexo 博客内容的存储库名称。

请确保在替换上述 URL 中的 username 和 repository 部分时使用正确的用户名和存储库名称。这个 URL 将指向您的 Hexo 博客在 GitHub Pages 上托管的页面。

自定义域名

阿里云域名购买流程

前提：注册阿里云账号（第三方登录支付宝和淘宝都行）+ 账号实名认证

  • 选择
    常见的后缀名很多，像com、cn、top、xyz、xin等等等等。选择适合自己的域名，要看域名的新注册费用和续费，综合对比，选择最合适的后缀名。对比下来感觉.top还算合适，续费一年甚至三年十年的价格都还能承受，而且首年只需1元。

  • 查询

    查询购买价格包括续费价格 查询是否被注册掉

  • 购买+实名认证

    查询好后，进入新用户首购福利进行购买，首年只要1元。购买时需要创建模板信息，创建完成后会发现没有进行实名，之前的实名只是个人信息的实名认证，还要进行购买域名的实名认证。点击“模板未实名”，根据提示进行填写相关信息，提交后会显示：此类审核需要42分钟，然后只需等待。

    如果没有实名认证，可以先购买，然后再认证。会给你一个公共模板，但是认证完要尽快过户。也可以等注册局审核完再认证。

    阿里云已完成初审，并已提交至注册局进行审核，实际审核完成时间以注册局结果为准。审核完成您的账号手机会收到短信通知，请您注意查收。根据注册局近期审核结果，该证件类型的审核时长约42分钟，请您耐心等待。一般一小时左右即可。

    第一次认证失败，身份证没有拍反面，所以提交身份证的时候，拼图成一张提交。

    最后进入阿里云的域名控制台，进行查看。完成购买之后还需要对新购买的域名进行解析，添加解析记录。

添加解析记录

进入阿里云控制台找到域名管理，点击“解析”。

进入解析设置页面——点击添加记录即可；要添加两个记录值，记录类型分别为 A 和 CNAME ，填写如下：

  • 第一个
    记录类型：A
    主机记录为：@
    记录值为自己git.io那个ip地址。IP地址在cmd中ping一下即可; 一般都是这个:185.199.110.153
  • 第二个
    记录类型：CNAME
    主机记录：www
    记录值为自己的github访问地址：用户名.github.io

再返回控制台发现解析设置页面添加了两条解析记录，状态均为正常。

到这里阿里云上的配置就结束了
